# 业务能力图 (Capability Graph) 设计文档

## 一、概述

基于代码图 (code-graph) 和语义图 (semantic-graph)，构建 L3-L6 业务能力层次图，用于需求变更影响分析。

```
语义图 (semantic-graph.json)
    │
    ▼
┌─────────────────────────────────────────────────────────────────┐
│                     业务能力图 (Capability Graph)               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  L3: 业务流程 (Process)                                        │
│      例: 支付下单流程                                           │
│      - 包含多个 L4 活动 + L5 规则                               │
│      - 引用多个 L6 类                                          │
│      - 有自己的流程图                                           │
│                                                                 │
│  L4: 活动/节点 (Activity)                                      │
│      例: IP白名单校验                                           │
│      - 单个活动的子流程                                         │
│      - 输入/输出/描述/流程图                                     │
│      - 包含 L5 规则                                             │
│      - 引用 L6 类                                               │
│      - 可以调用其他 L4                                          │
│                                                                 │
│  L5: 规则 (Rule)                                                │
│      例: 请求IP必须在白名单                                      │
│      - 从代码 if/else/for 提取                                  │
│      - 引用涉及的 L6 类                                         │
│                                                                 │
│  L6: 类 (Class)                                                 │
│      例: RpAccount                                              │
│      - Entity/PO/DTO/VO/BO                                      │
│      - 不等同于数据库表                                          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 二、Node 定义

### 2.1 共同属性

```python
class CapabilityNode:
    # 通用
    id: str              # 唯一标识，如 L3-001, L4-001, L5-001, L6-001
    name: str            # 名称
    level: int           # 层级 3-6
    type: str            # process/activity/rule/class
    
    # 元数据
    description: str     # 描述
    tags: list[str]       # 标签，如 ["支付", "安全"]
    
    # 溯源
    source: {
        "origin": "semantic-graph" | "code-graph" | "llm",
        "method": str,
        "file": str
    }
```

### 2.2 层级定义

| 层级 | 名称 | 描述 | 来源 |
|------|------|------|------|
| L3 | 业务流程 | 调用链，包含 L4 活动 + L5 规则 + 引用 L6 | 从入口方法调用链构建 |
| L4 | 活动/节点 | 单个方法的活动，包含流程图，可调用其他 L4 | 方法 (method) |
| L5 | 规则 | 业务约束，从代码 if/else/for 提取 | 从代码提取 |
| L6 | 类 | PO/DTO/VO/Entity/BO | 从代码 import/引用提取 |

| **contains** | 包含 (层级关系) | L3 contains L4, L3 contains L5, L4 contains L5 |

## 五、如何建立节点和边

### 5.1 节点建立

```
步骤1: L6 (类) ← 从代码提取
─────────────────────────────────────────────
- 遍历方法的参数类型
- 遍历方法内部 new 的对象类型
- 遍历 import 语句
- LLM 分类为 Entity/PO/DTO/VO/BO

步骤2: L5 (规则) ← 从代码提取
─────────────────────────────────────────────
- 遍历方法内的 if/else if 条件
- 遍历 for/while 循环约束
- 提取参数校验规则
- LLM 补充规则内容

步骤3: L4 (活动) ← semantic-graph.json
─────────────────────────────────────────────
- 每个方法 → L4
- 基础属性: description, input, output, flow, flow_chart
- LLM 补充 contains (L5) 和 references (L6)

步骤4: L3 (流程) ← 调用链
─────────────────────────────────────────────
- 从入口方法沿 CALL 边遍历
- 按调用顺序构建 L3
- contains = 包含的 L4 列表 + L5 列表
- references = 所有 L4 引用的 L6 合并
- flow_chart = 从各 L4 的 flow_chart 串联
```

### 5.2 边建立

```
步骤1: contains 边
─────────────────────────────────────────────
- L4 → L5: 从方法的 business_rules 自动建立
- L3 → L4: 按调用链顺序建立

步骤2: references 边
─────────────────────────────────────────────
- L4 → L6: 从方法参数类型、new 对象提取
- L5 → L6: 规则涉及的类

步骤3: calls 边
─────────────────────────────────────────────
- L4 → L4: 从 code-graph.json 的 CALL 边映射
```
